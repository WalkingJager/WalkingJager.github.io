关于Unity 2D Camera Size的一些理解

发表于 2016-03-24   |   分类于 Unity3d   |     |   阅读次数

通常我们在做用Unity做2D游戏时，会将摄像机的Projection设置为Orthographic（正交投影），在此模式下场景中的物体不会因为摄像机距离的远近而产生远大近小的效果，都是一样的大小。因此2D游戏中Main Camera默认设置如下图：


图中红色方框里的Position是2D摄像机的默认设置。Size定义了摄像机的视口大小，表示从屏幕的中心点到顶部的距离，单位是units。换而言之，就是视口高度的一半，而视口的宽度是通过aspect（宽高比）来计算：

2D中摄像机的Size大小决定了我们实际上可以看到游戏内容的多少，如果Size属性值等于当前屏幕高度units一半时，摄像机大小则正好与屏幕大小相等，这里的units是通过像素到单位比即Pixels To Units换算后的值。

问题：如何让背景图片正确的占满屏幕？

假设我们游戏的参考分辨率是1136x640（16：9），背景图片的大小为2048x640，则摄像机的Size大小为3.2

同理游戏的参考分辨率是640x1136（9：16），背景图片的大小为640x2048，则摄像机的Size大小为10.24
